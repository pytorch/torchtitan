# Nightly Scout Report — 2026-04-06

## Action Items (new findings)

- [ ] **[P0] `test_numerics.py` fully disabled in CI** — Both `TestSimpleFSDP`
  and `TestGraphTrainerNumerics` are commented out in both A10 and H100 CI
  workflows (disabled in commit `1b394fde` today). There is zero numerics
  regression testing in CI. The blocker is traced to
  `pytorch/pytorch#160509` (closed without merge). Need to investigate the
  actual root cause and find a path to re-enable, or track the upstream
  blocker explicitly.

- [ ] **[P1] `TestTraceModels`, `TestTraceFSDP`, `TestAutogradGradVsBackwardFSDP`
  not invoked in CI** — `test_trace_module.py` is in CI but the `-k` filter
  only runs `TestTraceModule`, `TestTraceDTensor`, and
  `TestMetadataPropagation`. Three test classes (covering single-GPU model
  tracing for llama3/qwen3/deepseek_v3/llama4/gpt_oss, multi-GPU FSDP
  tracing, and autograd.grad vs backward) are silently skipped. Add them to
  the CI `-k` filter in the A10 workflow.

- [ ] **[P1] `run_precompile_tests.py` never invoked in CI** — This test runner
  defines a full end-to-end precompile+training flow
  (`aot_llama3_precompile_full_inductor`) but is not invoked in either CI
  workflow. The entire CooR precompile feature has no automated CI coverage.

- [ ] **[P1] 9 of 11 graph passes lack unit tests** — Only `apply_sac_pass`
  and `reassign_to_pg_pass` have dedicated unit tests. The following passes
  have zero unit test coverage: `auto_bucketing`,
  `transformer_block_bucketing`, `regional_inductor`, `cudagraph`,
  `full_inductor_compilation`, `inductor_decomposition`,
  `fsdp_reshard_after_fwd`, `validate_flex_attn_annotation`,
  `apply_default_graph_passes`. Some are exercised via integration tests
  but most have no isolated test.

- [ ] **[P2] `ParallelDims` manual construction in `precompile_main.py`** —
  `precompile_main.py:118-127` manually constructs `ParallelDims(...)`.
  The new `ParallelDims.from_config()` classmethod (commit `a146a4de`)
  could simplify this. Minor cleanup.

## Carried Forward (from prior reports, re-investigated)

- [ ] **[P0] Llama3 `apply_tp` missing `enable_cp` and `enable_sp`** — First
  reported 2026-04-02. **Still valid.** `llama3/parallelize.py:99-104`
  calls `apply_tp()` without `enable_cp` or `enable_sp`. DeepSeek-v3's
  parallelize correctly passes both. CP is silently non-functional for
  llama3 in graph_trainer. A commit on the `self_improve/2026-04-03` branch
  does NOT address this. README still marks CP as ✅.

- [ ] **[P0] `fsdp_reshard_after_fwd_pass` has zero test coverage** — First
  reported 2026-04-02. No change. This joint pass is registered in
  `AVAILABLE_JOINT_PASSES` but has no unit test and no integration test.

- [ ] **[P1] `to_graph_trainer_config()` silently discards AC sub-fields** —
  First reported 2026-04-02. `configs.py:74-76` creates a fresh
  `ActivationCheckpointConfig(mode="selective")` losing user-configured
  sub-fields. Currently inert (graph SAC doesn't consume them) but will
  silently drop user intent if these fields are ever wired up.

- [ ] **[P1] SAC pass diverged from core** — First reported 2026-04-02. Three
  divergences remain: (1) missing CUDA-to-CPU `_to_copy` as MUST_SAVE,
  (2) missing `per_op_sac_force_recompute_mm_shapes_by_fqns` support,
  (3) missing `aten.linear.default` in mm_ops. Items (1) and (2) are
  relevant for DeepSeek-v3 MoE workloads.

- [ ] **[P1] `precompile` has no end-to-end integration test** — First reported
  2026-04-02. `run_precompile_tests.py` exists but is not in CI (also
  flagged as new P1 above). Unit tests cover components but not the full
  compile-save-reload-train loop.

- [ ] **[P2] DeepSeek-v3 missing pass parity with Llama3** — First reported
  2026-04-02. No change. DeepSeek-v3 has no AOT integration tests for
  `auto_bucketing`, `cudagraph`, `regional_inductor`, or
  `full_inductor_compilation`.

- [ ] **[P2] Gradient accumulation untested** — First reported 2026-04-02. No
  change. Core tests custom `global_batch_size` for gradient accumulation
  but graph_trainer never tests this.

- [ ] **[P2] README.md doesn't mention `aot_fx_trace` mode** — First reported
  2026-04-02. **Still unfixed on main.** README line 12 says "Two
  compilation modes" but there are three (`jit`, `aot`, `aot_fx_trace`).
  A fix exists on the `self_improve/2026-04-03` branch but hasn't been
  merged to main.
