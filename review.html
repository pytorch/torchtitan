Summary

Converts the Qwen3-VL vision encoder submodules (PatchEmbed, VisionRotaryEmbedding, PatchMerger, VisionAttention, VisionMLP, VisionTransformerBlock, Qwen3VLVisionEncoder) to the Module protocol with proper Config subclasses. Adds set_qwen3_vl_sharding_config (with a private _set_vision_encoder_sharding) that fills both decoder dense and vision encoder sharding via ShardingConfig on Config instances. Deletes the legacy _apply_tp_to_vision_encoder helper and the now-unused distribute_tensor/ColwiseParallel/RowwiseParallel/NoParallel imports from parallelize.py. Wires update_from_config on Qwen3VLModel.Config to call set_qwen3_vl_sharding_config.

Round 3 adds an explicit "TP > 1 with the vision encoder is unverified end-to-end" caveat in the public set_qwen3_vl_sharding_config docstring; FSDP-only is verified per MEMORY.md.

Analysis

Module protocol conversion (vision_encoder.py). All seven classes follow the same pattern: @dataclass(kw_only=True, slots=True) class Config(Module.Config) + __init__(self, config: Config) that builds child configs. The conversion matches the convention used by Linear, RMSNorm, Embedding, and the newly added nn_modules.py wrappers from PR7. The PatchMerger handling of the per-norm normalized_shape swap via dataclasses.replace preserves the shared sharding_config on the norm Config (shallow copy semantics are correct here because the placement dict is intentionally shared).

Norm Config sharing across layers. norm1=config.block_norm, norm2=config.block_norm on every VisionTransformerBlock (line 571-572) intentionally shares one LayerNorm.Config instance, so a single sharding_config assignment in _set_vision_encoder_sharding propagates to all n_layers * 2 norms. Same approach for merger_norm shared across merger + all deepstack_merger_list entries. Each .build() call constructs a fresh LayerNorm module so there's no module aliasing.

Sharding placements (sharding.py). The vision encoder uses an all-Replicate activation discipline (no SP), with colwise+rowwise linear pairs that all-reduce back to Replicate at the pair boundary. The new _vision_colwise_config sensibly diverges from decoder_sharding.colwise_config by setting out_dst_shardings=Replicate (instead of Shard(-1)) because vision encoder activations are always Replicate between modules. _replicate_output_rowwise_config declares in_src/in_dst=Shard(-1) so DTensor inserts the chunk before the rowwise input. Norm weight/bias placements are Replicate, matching the standard plan. pos_embed is declared on the encoder-root state_shardings as Replicate on TP, mirroring the legacy plan.

Equivalence to the deleted legacy plan. The new placement plan differs in one observable way: the legacy attn.qkv used ColwiseParallel(use_local_output=True) so flex_attention ran on the local heads on each rank; the new path forces an all-gather back to Replicate after attn.qkv, so attention runs on the full head set on every rank. Memory savings on the linear weights are preserved; compute on attention is redundant under TP. Both the parent and private docstrings flag this explicitly. Restoring heads-parallel attention would require a local_map on VisionAttention (analogous to set_gqa_inner_attention_local_map in decoder_sharding.py); marked as future work, acceptable scope decision.

Parallelize flow (parallelize.py). Deleting _apply_tp_to_vision_encoder and routing through model.parallelize(parallel_dims) matches the unified dense-sharding flow from the rest of the stack. The unused TP imports were correctly cleaned up. The remaining maybe_enable_async_tp import is still used.

__init__.py wiring. Adds block_norm and merger_norm as LayerNorm.Config(normalized_shape=dim, eps=1e-6). The merger_norm's normalized_shape is later swapped to merged_hidden_size for postshuffle DeepStack mergers inside PatchMerger.__init__ via dataclasses.replace, which is the right pattern.

TP > 1 verification caveat. Round 3's added paragraph in set_qwen3_vl_sharding_config ("TP > 1 with the vision encoder is unverified end-to-end") matches the MEMORY.md state (FSDP-only verified). Note this caveat lives in the parent (public) docstring rather than in _set_vision_encoder_sharding as the task description suggested. Either placement is defensible -- the public one is arguably more visible to callers in update_from_config.
